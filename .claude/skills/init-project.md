# Skill: /init-project — Project scaffold from docs

## Description
Reads `docs/tech-stack.md` and sets up the bare project skeleton: framework init, dependency install, folder structure, and config files. Does NOT create any business logic, pages, services, or API routes — that's `/plan` + `/implement`.

## Usage
```
/init-project          — Full scaffold
/init-project dry-run  — Show what would be created without doing it
```

## Prerequisites
- `docs/tech-stack.md` must exist. If missing, suggest `/docs` first.

## Instructions

### Step 1: Read tech-stack.md
Extract: framework, dependencies list, folder structure, package manager.

If argument is `dry-run`, show summary and stop.

### Step 2: Init framework
```bash
pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --no-turbopack --import-alias "@/*" --use-pnpm
```
If `package.json` already exists, ask the user before continuing.

### Step 3: Install dependencies
From tech-stack.md. Typical set:
```bash
pnpm add axios @tanstack/react-query @supabase/supabase-js @supabase/ssr zod date-fns
pnpm dlx shadcn@latest init -d
pnpm dlx shadcn@latest add button card input label separator badge textarea
```

### Step 4: Create folder structure
Only empty directories that don't exist yet:
```
src/
├── services/
├── components/providers/
├── lib/supabase/
├── types/
├── app/api/auth/callback/
├── app/(public)/login/
└── app/(protected)/
```

### Step 5: Create config files only
These are infrastructure files, NOT business logic:

| File | What it does |
|------|-------------|
| `src/lib/env.ts` | Zod validator for env vars (lazy proxy) |
| `src/lib/api-client.ts` | `genericAuthRequest<T>(method, url, data)` with axios + auth interceptor |
| `src/lib/query-hooks.ts` | `useAppQuery<T>` and `useAppMutation<TInput, TOutput>` wrappers |
| `src/lib/query-keys.ts` | Empty `queryKeys` object (entities added later by `/implement`) |
| `src/lib/supabase/client.ts` | Lazy browser client via proxy |
| `src/lib/supabase/server.ts` | Server client with cookies |
| `src/lib/auth.ts` | `getAuthUser(req)`, `getSupabaseForUser(req)`, `unauthorized()` |
| `src/components/providers/query-provider.tsx` | QueryClientProvider wrapper |
| `src/proxy.ts` | Route protection (redirect to /login if no session) |
| `src/types/index.ts` | Empty file with `// Types will be generated by /implement` |
| `.env.local.example` | Template with required env var names |

### Step 6: Verify
1. Run `pnpm build` — must compile clean
2. Show summary: files created, deps installed, next step is `/plan`

## What this skill does NOT do
- No pages (no dashboard, no forms, no lists)
- No services (no CRUD hooks)
- No API routes (no business endpoints)
- No types/DTOs (only empty placeholder)
- No data model implementation

Those are created by `/plan` → `/implement`.

## MANDATORY Conventions (from CLAUDE.md)
- **kebab-case** for all files and folders
- **English** for all code
- Route groups: `(public)` and `(protected)`
- Env vars only through `src/lib/env.ts`
